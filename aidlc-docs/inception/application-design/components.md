# コンポーネント定義

## TUI層

### App（メインアプリケーション）

- **目的**：Textual Appのエントリポイント。画面レイアウト管理、グローバルキーバインド、画面遷移を担う
- **責任**：
  - 左ペイン・右ペインのレイアウト構成
  - グローバルキーバインドの処理（ヘルプ表示、終了等）
  - CLI引数の解析とMR直接展開の制御
  - アプリケーションライフサイクル管理

### MRListPanel（左ペイン - MR一覧パネル）

- **目的**：MRをカテゴリ別にツリー表示し、選択操作を受け付ける
- **責任**：
  - 4カテゴリ（自身割当/自身作成/未割当/他者割当）のMRツリー表示
  - MR選択時のツリー展開（Overview選択肢 + 変更ファイル一覧）
  - 選択変更イベントの発行
  - MR一覧のリフレッシュ

### ContentPanel（右ペイン - コンテンツ表示パネル）

- **目的**：選択されたMRのOverviewまたは差分を表示する
- **責任**：
  - MR Overview表示（タイトル、説明、ステータス、ディスカッション等）
  - ファイル差分表示（Pygmentsによるシンタックスハイライト付き）
  - 差分内の行選択とコメント挿入トリガー
  - 既存コメントのインライン表示

### CommentDialog（コメント入力ダイアログ）

- **目的**：コメントの入力・送信を行うモーダルダイアログ
- **責任**：
  - テキスト入力エリアの提供
  - インラインコメント / ノート / リプライの種別管理
  - コメント送信の確認と実行

### HelpScreen（ヘルプ画面）

- **目的**：キーバインド一覧と使い方を表示するオーバーレイ
- **責任**：
  - キーバインド一覧の表示
  - コンテキスト別のヘルプ情報提供

## API層

### GitLabClient

- **目的**：python-gitlabライブラリのラッパー。認証とAPI接続を管理する
- **責任**：
  - パーソナルアクセストークンによる認証
  - GitLabインスタンスへの接続管理
  - プロジェクトIDの解決
  - エラーハンドリングとリトライ

### MRService

- **目的**：MR関連のビジネスロジックをカプセル化する
- **責任**：
  - MR一覧の取得とカテゴリ分類
  - MR詳細情報の取得
  - MR変更ファイル一覧の取得
  - ファイル差分の取得

### CommentService

- **目的**：コメント関連のビジネスロジックをカプセル化する
- **責任**：
  - MRのディスカッション/ノート一覧の取得
  - インラインコメントの投稿
  - MR全体へのノートの投稿
  - 既存コメントへのリプライ

## 基盤層

### ConfigManager

- **目的**：設定ファイルの読み書きと設定値の提供
- **責任**：
  - TOML設定ファイルの読み込み（~/.config/lazygitlab/config.toml）
  - デフォルト値の提供
  - 設定値の検証
  - 初回実行時の設定ファイルテンプレート生成

### GitRepoDetector

- **目的**：カレントディレクトリからGitLabプロジェクト情報を検出する
- **責任**：
  - gitリモートURL（origin）の取得
  - SSH/HTTPS URL形式の解析
  - GitLabホスト名とプロジェクトパスの抽出
  - プロジェクトIDへの変換（API経由）

### Logger

- **目的**：構造化ロギングの提供（SECURITY-03準拠）
- **責任**：
  - 構造化ログ出力（JSON形式）
  - ログレベル制御
  - 機密データ（トークン等）のマスキング
  - ファイルへのログ出力（~/.local/share/lazygitlab/logs/）
