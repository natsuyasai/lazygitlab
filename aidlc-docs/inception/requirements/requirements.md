# 要件ドキュメント - lazygitlab

## 意図分析サマリー

- **ユーザーリクエスト**：ターミナル上でGitLabのマージリクエスト（MR）を閲覧し、コメントの追加を行えるTUIソフトウェアの作成
- **リクエストタイプ**：新規プロジェクト（グリーンフィールド）
- **スコープ見積もり**：システム全体（新規TUIアプリケーション）
- **複雑さ見積もり**：複雑（GitLab API連携、マルチペインTUI、差分表示、コメント機能）

## 機能要件

### FR-01：MR一覧表示

- 起動時に左ペインで以下のカテゴリに分類してMRを表示する：
  - 自身に割り当てられているMR
  - 自身が作成したMR
  - 未割当のMR
  - 自身以外に割り当てられているMR
- カレントディレクトリのgitリモートURLからGitLabプロジェクトを自動検出する

### FR-02：MRツリー展開

- MRを選択しツリーを展開すると以下が表示される：
  - Overview表示の選択肢
  - 変更ファイルの一覧

### FR-03：Overview表示

- 右ペインにMRのOverview情報を表示する：
  - タイトル、説明、作成者、アサイニー
  - ステータス、ラベル、マイルストーン
  - パイプライン状況
  - 既存のディスカッション/コメント

### FR-04：差分表示

- 右ペインに変更ファイルの差分を表示する
- unified diff形式またはside-by-side形式での表示
- シンタックスハイライト対応（Pygments等のライブラリによるトークンベースハイライト）

### FR-04a：外部エディタ連携

- 差分表示中にキー操作で任意のターミナルエディタ（vim, nano, emacs等）を開くオプション
- 設定ファイルで使用するエディタを指定可能（デフォルト：$EDITOR環境変数）
- エディタで該当ファイルの該当行にジャンプして開く

### FR-05：コメント機能

- 差分表示中に行を選択してインラインコメントを挿入可能
- MR全体へのノート（概要コメント）を追加可能
- 既存コメントへのリプライが可能

### FR-06：引数によるMR指定起動

- コマンドライン引数でMRのIDを指定可能
- 指定された場合、即座にそのMRを展開した状態で起動する

### FR-07：プロジェクト自動検出

- カレントディレクトリのgitリモートURL（origin）を解析してGitLabプロジェクトIDを特定する
- SSH/HTTPS両方のリモートURL形式に対応する

## 非機能要件

### NFR-01：開発言語・技術スタック

- **言語**：Python
- **TUIフレームワーク**：textualまたは同等のPython TUIライブラリ
- **GitLab API**：python-gitlab または REST API直接呼び出し
- **配布**：pip install / pipx install で導入可能

### NFR-02：認証

- パーソナルアクセストークンによる認証
- 設定ファイル（~/.config/lazygitlab/config.toml等）でトークンとGitLab URLを管理
- トークンはファイルシステムのパーミッションで保護（0600推奨）

### NFR-03：対応GitLab

- セルフホスト型GitLabとgitlab.com（SaaS）の両方に対応
- GitLab URLを設定で切り替え可能

### NFR-04：操作スコープ

- 閲覧とコメントのみ（MRのステータス変更、承認、マージ、クローズ等の操作は対象外）

### NFR-05：パフォーマンス

- MR一覧の初回取得は5秒以内を目標
- キャッシュまたは遅延読み込みによるレスポンス改善

### NFR-06：セキュリティ

- セキュリティベースラインルール（SECURITY-01〜15）を適用
- トークンをログに出力しない（SECURITY-03準拠）
- 設定ファイルにハードコードされた認証情報を含めない（SECURITY-12準拠）
- エラーメッセージに内部詳細を含めない（SECURITY-09準拠）
- 依存関係のバージョンをピン留めする（SECURITY-10準拠）
- 構造化ロギングを使用する（SECURITY-03準拠）
- 外部API呼び出しにエラーハンドリングを実装する（SECURITY-15準拠）

### NFR-07：ユーザビリティ

- lazygit風のキーバインド体系
- ヘルプ画面（キーバインド一覧）の表示
- ターミナルサイズの変更に追従するレスポンシブレイアウト

## 対象外（スコープ外）

- MRの承認、マージ、クローズ等のステータス変更操作
- 複数プロジェクトの横断表示
- CI/CDパイプラインの詳細操作
- Issue管理
- コードレビューのリゾルブ/アンリゾルブ操作
