# UNIT-01: Infrastructure NFR要件

## 1. パフォーマンス要件

### PERF-01: 設定ファイル読み込み

| 項目 | 値 |
|---|---|
| 目標 | 100ms以内 |
| 根拠 | TUI起動時のブロッキング処理。ユーザー体感に影響 |
| 測定方法 | ConfigManager.load() の実行時間 |

### PERF-02: gitコマンド実行タイムアウト

| 項目 | 値 |
|---|---|
| タイムアウト | 30秒 |
| 根拠 | 低速ネットワーク環境（VPN経由等）を考慮 |
| 適用対象 | subprocess.run() によるgitコマンド呼び出しすべて |
| タイムアウト時の挙動 | subprocess.TimeoutExpired を捕捉し、ユーザーにエラーメッセージを表示 |

### PERF-03: ログ書き込みオーバーヘッド

| 項目 | 値 |
|---|---|
| 目標 | ログ書き込みがメインスレッドをブロックしない |
| 実装方針 | 標準logging（同期）で十分。TUIのイベントループ外で実行される基盤層の処理のため |

## 2. セキュリティ要件

### SEC-01: トークン保管（SECURITY-12準拠）

| 項目 | 値 |
|---|---|
| 保管場所 | `~/.config/lazygitlab/config.toml` |
| ファイルパーミッション | 0600（Unix系のみ） |
| ハードコード禁止 | ソースコードにトークンを含めない |
| セットアップウィザード | トークン入力時に画面表示をマスクする |

### SEC-02: ログへの機密情報漏洩防止（SECURITY-03準拠）

| 項目 | 値 |
|---|---|
| マスク対象 | `glpat-`で始まる文字列、20文字以上の英数字連続列 |
| 置換文字列 | `***REDACTED***` |
| 適用範囲 | ログメッセージ本文および引数 |
| 実装 | TokenMaskingFilter（logging.Filter サブクラス） |

### SEC-03: エラーメッセージの制限（SECURITY-09準拠）

| 項目 | 値 |
|---|---|
| ユーザー向けメッセージ | 汎用的なエラー説明のみ |
| 内部詳細 | スタックトレースはDEBUGレベルのログにのみ出力 |
| トークン | エラーメッセージにトークン値を含めない |
| パス情報 | フルパスの代わりにファイル名のみ表示 |

### SEC-04: 例外処理（SECURITY-15準拠）

| 項目 | 値 |
|---|---|
| 外部呼び出し | すべてのsubprocess呼び出し、ファイルI/Oに明示的なtry/except |
| フェイルクローズド | エラー時はアプリケーション起動を停止（不完全な状態で起動しない） |
| リソースクリーンアップ | ファイルハンドルはwith文で管理 |
| グローバルハンドラ | __main__.py にトップレベルのtry/except で未処理例外をキャッチ・ログ記録 |

### SEC-05: 依存関係管理（SECURITY-10準拠）

| 項目 | 値 |
|---|---|
| バージョン固定 | pyproject.toml で正確なバージョンまたは互換範囲を指定 |
| ロックファイル | pip-compile等でロックファイルを生成・コミット |
| 未使用依存関係 | 積極的に使用されないパッケージを含めない |
| ソース | PyPI（公式レジストリ）のみ |

## 3. 信頼性要件

### REL-01: gitコマンド失敗時の挙動

| エラー種別 | 挙動 |
|---|---|
| gitコマンド未インストール | `GitCommandNotFoundError` を発生、明確なメッセージで終了 |
| gitリポジトリ外で実行 | `NotAGitRepositoryError` を発生、メッセージ表示して終了 |
| リモート未設定 | `NoRemoteConfiguredError` を発生、メッセージ表示して終了 |
| タイムアウト（30秒超過） | タイムアウトメッセージを表示して終了 |
| その他のsubprocessエラー | エラーメッセージを表示、詳細はDEBUGログに出力 |

### REL-02: 設定ファイル破損時のリカバリ

| 条件 | 挙動 |
|---|---|
| TOMLパースエラー | パースエラーの概要を表示（行番号含む） |
| リカバリ提案 | セットアップウィザードの再実行を提案 |
| ユーザー選択 | ウィザード再実行 or 手動修正（終了）を選択可能 |

### REL-03: ファイルシステムエラー

| エラー種別 | 挙動 |
|---|---|
| 設定ディレクトリ作成失敗 | パーミッションエラーメッセージを表示して終了 |
| ログディレクトリ作成失敗 | 警告を表示し、ログなしで続行 |
| ログファイル書き込み失敗 | 警告をstderrに出力し、ログなしで続行 |

## 4. 保守性要件

### MAINT-01: テスト戦略

| テストレベル | 方針 | 対象 |
|---|---|---|
| 単体テスト | モック中心 | ConfigManager（ファイルI/Oモック）、GitRepoDetector（subprocessモック）、Logger（ハンドラモック） |
| 統合テスト | 実環境 | 一時ディレクトリにgitリポジトリを作成、実際のTOMLファイル読み書き、ログファイル出力確認 |
| テストフレームワーク | pytest | |
| カバレッジ目標 | 80%以上 | |

### MAINT-02: コード品質

| 項目 | 値 |
|---|---|
| 型ヒント | すべてのパブリックAPIに型ヒントを付与 |
| リンター | ruff |
| フォーマッター | ruff format |

### MAINT-03: ログ運用

| 項目 | 値 |
|---|---|
| 保持期間 | 30日間 |
| ローテーション | 5MB/ファイル、3世代保持 |
| 古いログの削除 | アプリ起動時に30日以上前のログファイルを自動削除 |
