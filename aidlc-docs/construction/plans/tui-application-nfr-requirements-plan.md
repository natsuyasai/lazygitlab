# UNIT-03: TUI Application NFR要件計画

## 概要

UNIT-03（TUI Application）の非機能要件を評価する。ローカルCLI/TUIアプリケーションのUI層として、パフォーマンス、ユーザビリティ、信頼性、テスト戦略に焦点を当てる。

## 評価ステップ

- [x] 1. パフォーマンス要件の評価
  - MR一覧初回表示までの応答時間
  - 差分表示のレンダリング性能
  - 大きなMR（多数ファイル、大きな差分）の扱い

- [x] 2. ユーザビリティ要件の評価
  - ターミナルサイズ対応
  - ローディング表示
  - フォーカス管理

- [x] 3. 信頼性要件の評価
  - API呼び出し失敗時のUI復帰
  - 外部エディタ連携のエラーハンドリング

- [x] 4. 保守性要件の評価
  - テスト戦略（Textualのテスト方法）
  - コードカバレッジ目標

- [x] 5. テックスタック決定
  - Textual / Pygments バージョン固定
