# UNIT-02: GitLab API Services 機能設計計画

## 概要

UNIT-02（GitLab API Services）の機能設計を行う。python-gitlabを通じたGitLab API通信、MR一覧・詳細・差分取得、およびコメント操作のビジネスロジックを設計する。

## 計画ステップ

- [ ] ステップ1：GitLabClient のビジネスロジック設計（認証、接続管理、プロジェクト解決）
- [ ] ステップ2：MRService のビジネスロジック設計（カテゴリ別一覧、詳細、差分取得）
- [ ] ステップ3：CommentService のビジネスロジック設計（ディスカッション取得、コメント投稿）
- [ ] ステップ4：ドメインエンティティの詳細設計（python-gitlab オブジェクトから内部モデルへのマッピング）
- [ ] ステップ5：ビジネスルール・バリデーションロジックの定義
- [ ] ステップ6：エラーハンドリング戦略の設計
