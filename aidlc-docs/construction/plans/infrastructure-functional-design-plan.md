# UNIT-01: Infrastructure 機能設計計画

## 概要

UNIT-01（基盤層）の詳細なビジネスロジック設計を行う。
対象コンポーネント：ConfigManager、GitRepoDetector、Logger、データモデル

## 設計ステップ

- [x] 1. ConfigManager ビジネスロジック設計
  - TOML設定ファイルの構造定義
  - デフォルト設定値の決定
  - バリデーションルールの定義
  - 設定ファイルの自動生成ロジック
  - ファイルパーミッション管理（SECURITY-12準拠）

- [x] 2. GitRepoDetector ビジネスロジック設計
  - gitリモートURL解析アルゴリズム（SSH/HTTPS対応）
  - GitLabホスト・プロジェクトパスの抽出ロジック
  - エラーハンドリング（git未初期化、remote未設定等）

- [x] 3. Logger ビジネスロジック設計
  - 構造化ロギングのセットアップロジック
  - トークンマスキングルール（SECURITY-03準拠）
  - ログレベル制御

- [x] 4. データモデル詳細設計
  - 各dataclassのフィールド定義とバリデーション
  - エンティティ間の関係定義
  - 型制約とデフォルト値

- [x] 5. ビジネスルール定義
  - 設定ファイル関連のバリデーションルール
  - URL解析の正規化ルール
  - セキュリティ関連の制約
