# UNIT-03: TUI Application NFR要件

## 概要

UNIT-03（TUI Application）の非機能要件を定義する。ローカルCLI/TUIアプリケーションのUI層として、パフォーマンス、ユーザビリティ、信頼性、保守性の観点で要件を規定する。

---

## 1. パフォーマンス要件

### NFR-TUI-PERF-01: MR一覧初回表示

- MR一覧の初回表示までの応答時間は5秒以内を目標とする（NFR-05準拠）
- UNIT-02の `asyncio.gather` による4カテゴリ並行取得を活用する
- ステータスバーにローディングインジケーターを表示し、ユーザーに待ち時間を明示する

### NFR-TUI-PERF-02: 差分表示のレンダリング

- 通常サイズの差分（500行以下）は1秒以内にレンダリングを完了する
- Pygmentsによるシンタックスハイライトを含むレンダリングコストを考慮する
- 差分表示時はステータスバーにローディングインジケーターを表示する

### NFR-TUI-PERF-03: 大きな差分の仮想スクロール

- 大きな差分ファイル（数千行レベル）は**仮想スクロール**方式で表示する
- 画面に見える範囲のみレンダリングし、スクロールに応じて追加描画する
- Textualの `ScrollableContainer` / `RichLog` のスクロール機構を活用する
- メモリ使用量は差分全体をパース済みデータとして保持するが、レンダリングは表示範囲に限定する

---

## 2. ユーザビリティ要件

### NFR-TUI-UX-01: ターミナルサイズ対応

- 最小ターミナルサイズ: 80列 × 24行
- ターミナルリサイズ時にTextualの自動レイアウト再計算を利用する
- 左ペイン（30%）と右ペイン（70%）の比率はターミナル幅に追従する
- サイドバートグル（`[` キー）により小さいターミナルでもコンテンツ領域を最大化可能

### NFR-TUI-UX-02: ローディング表示

- データ読み込み中は**ステータスバー（Footer領域）にインジケーター**を表示する
- コンテンツ領域の既存表示はローディング中もそのまま保持する（フリッカー防止）
- ローディング完了後、コンテンツを即座に差し替える

### NFR-TUI-UX-03: フォーカス管理

- アプリ起動時は左ペイン（MRListPanel）にフォーカスを設定する
- Tabキーで左ペイン ↔ 右ペイン間のフォーカス切替を行う
- モーダルダイアログ（CommentDialog, HelpScreen, ErrorDialog）表示中はモーダル内にフォーカスを閉じ込める
- モーダル閉じた後は元のフォーカス位置に復帰する

---

## 3. 信頼性要件

### NFR-TUI-REL-01: API呼び出し失敗時のUI復帰

- API呼び出し失敗時はErrorDialogでエラーメッセージを表示する
- エラーダイアログを閉じた後、アプリケーションは操作可能な状態を維持する
- 直前のキャッシュ済みデータがあれば、それを表示し続ける
- ユーザーの `r`（リフレッシュ）操作で再取得を試みる

### NFR-TUI-REL-02: 外部エディタ連携のエラーハンドリング

- エディタコマンドが見つからない場合はErrorDialogで通知する
- エディタプロセスが異常終了（非ゼロ終了コード）した場合もErrorDialogで通知する
- テンポラリファイルは正常/異常を問わず、処理完了後に必ず削除する（finally句）
- エディタ連携失敗後もCommentDialogは閉じず、TextAreaでの直接入力にフォールバック可能

### NFR-TUI-REL-03: 予期しない例外のハンドリング

- TUI層で発生する予期しない例外はキャッチし、ErrorDialogで表示する
- アプリケーション全体のクラッシュを防止する
- 例外詳細はUNIT-01のLoggerでログファイルに記録する

---

## 4. 保守性要件

### NFR-TUI-MAINT-01: テスト戦略

- **ロジック分離テスト（基本）**: ビジネスロジックのみ単体テストする
  - ツリーノードデータの構築ロジック
  - 差分パース・フォーマットロジック
  - コメントコンテキスト生成ロジック
  - 状態遷移ロジック（ContentViewState）
- **Textual Pilotテスト（基本ケース）**: `textual.testing` を使ったヘッドレステスト
  - アプリの起動・終了
  - キーバインドによる基本操作（サイドバートグル、フォーカス切替）
  - モーダルダイアログの表示・非表示
  - ウィジェットの初期状態確認

### NFR-TUI-MAINT-02: コードカバレッジ

- ビジネスロジック（UI非依存部分）のカバレッジ目標: 80%以上
- Textual Pilotテストは基本動作確認を目的とし、カバレッジ目標からは除外する
- 差分パース・フォーマットロジックは100%カバレッジを目指す

### NFR-TUI-MAINT-03: ログ出力

- UNIT-01のLogger（構造化ログ）を使用する
- TUI層のログレベル:
  - DEBUG: ユーザー操作のイベント（キー押下、ノード選択）
  - INFO: 画面遷移、外部エディタ起動
  - WARNING: 仮想スクロール境界のフォールバック
  - ERROR: UI復帰エラー、エディタ連携失敗
